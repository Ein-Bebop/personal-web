# Personal Website

## Project context
Personal portfolio/website. Prioritize clean, semantic HTML and modern CSS (not fancy stuff, more minimalist). I can upload this to a host.
No heavy frameworks unless really necessary.
Two main features, a blog section, where only two rows, one for date, other for title (link to the post), and a home section with information about me. There I want to tell who I am, where I have worked, etc... Lastly, the post page have title, date, post (can contain photos). I would like to be able to upload new posts using .MD files to a folder.


## Conventions
- Use kebab-case for file names
- Keep CSS in separate files, not inline
- Mobile-first responsive design
- Commit after each completed feature

## What not to do
- Don't install large JS frameworks without asking first
- Don't push directly to main
- Don't add features like filters, social media, sorting of posts, contact forms etc...
